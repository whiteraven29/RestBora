# RestBora V1 Product Scope

## Goal

Build a practical restaurant management MVP for local Tanzanian restaurants. V1 must prove the daily operating flow:

Customer or waiter creates order -> kitchen prepares order -> waiter serves order -> cashier confirms payment -> owner sees reports.

## V1 Users

- Owner/Admin
- Manager
- Waiter
- Kitchen staff
- Cashier
- Customer using public QR ordering

## V1 Core Modules

- Authentication and staff roles
- Restaurant profile
- Menu categories and items
- Table and QR code management
- Waiter order creation
- Public QR customer ordering
- Kitchen display
- Cashier payment tracking
- Owner dashboard
- Manual inventory management
- Expense tracking
- Estimated profit/loss
- Customer records and basic loyalty

## Explicitly Later: V2/Premium

- Payment gateway integration
- WhatsApp/SMS notifications
- Advanced loyalty automation
- Recipe-based automatic stock deduction
- Multi-branch operations in the UI
- Delivery rider tracking
- Customer reviews
- Coupons and offers
- Native mobile apps
- AI sales recommendations
- Accounting integration
- Offline mode

## Success Criteria

V1 is successful when a restaurant can run one full service day through the system: create menu items, receive orders, prepare food, record payment, track unpaid orders, record expenses and stock, then see a clear daily sales and estimated profit/loss summary.
