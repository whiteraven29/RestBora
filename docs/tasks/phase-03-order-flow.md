# Phase 3: Order Flow

## Objective

Harden the daily order lifecycle across waiter, customer, kitchen, and cashier.

## Tasks

- Add role-based access to waiter, kitchen, cashier, and owner screens.
- Add order cancellation with required reason.
- Add active order filters by status.
- Add clearer kitchen columns for pending, preparing, and ready.
- Add receipt view for paid orders.
- Add order status polling or lightweight real-time updates.

## Done When

Staff can run a normal service period without relying on admin screens or manual database edits.
